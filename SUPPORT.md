# Support Policy

Codex Swarm currently targets technically capable self-hosting teams.

The intended release-1 support boundary is:

- private self-hosted deployment
- single-host managed topology
- optional same-host worker fan-out

Not part of the supported release-1 boundary:

- public internet browser deployment
- generalized remote-worker fleet onboarding
- managed SaaS operation

Use the checked-in operational docs for:

- installation and deployment
- upgrade path
- backup and restore
- troubleshooting and operator workflows

Version support expectations are documented in
`docs/operations/supported-versions.md`.
