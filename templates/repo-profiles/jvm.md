# JVM Profile

## Use When

- the target repo is a Java or Kotlin service, worker, or library

## Baseline Layout

- Gradle or Maven wrapper committed
- reproducible build and test commands
- module boundaries documented
- CI entrypoints for compile, test, and packaging

## Swarm Defaults

- leader: milestone sequencing and integration checkpoints
- architect: module boundaries and build graph changes
- implementer: application and API slice delivery
- tester: integration and contract test validation

## Recommended Skills

- `plan-from-spec`
- `create-task-dag`
- `validate-milestone`
- `prepare-pr`
